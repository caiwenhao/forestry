**综述**
您可以使用数据概览模块了解店铺运营数据，以便调整运营策略。

**时间筛选**
该模块默认展示昨日数据。
如果需要查看其他日期范围的数据，可以点击日期选择器变更。
展示的数据包括当前周期和上一个周期的数据。

**指标说明**

| 指标 | 说明 |
| --- | --- |
| 销售额 | 已付款在线订单总销售额与COD订单销售额之和 |
| 访客数 | 访问店铺的总独立访客数（UV），按天去重，1个客户某1天访问多次也只统计1次。 |
| 订单数 | 统计顾客下单的数量（包括已付款在线订单与COD订单），1个顾客下多个订单则统计多次。 |
| 平均订单金额 | 销售额 / 订单数 |
| 购物行为 | 顾客的购物漏斗转化表现，包括：总转化率、加入购物车访客数和转化率、到达结账页访客数和转化率、添加收货信息访客数和转化率、选择物流方式访客数和转化率、选择支付方式访客数和转化率、完成结账访客数和转化率。 |
| 来源国家/地区 | 通过访客IP识别所在的国家/地区。 |
| 热销商品 | 展示销量最多的10个商品对应的数据。 |
| 着陆页浏览量 | 某个页面的PV |
| 流量来源 |   按流量来源分组展示对应的数据（包括：直接、社交、搜索、Email、其他）。 |
| 社交来源 | 按社交流量来源分组展示对应的数据（包括：Facebook、Instagram、Tiktok、Twitter、YouTube）。 |
| 设备来源 | 按设备来源分组展示对应的数据（包括：PC、移动、平板、其他）。 |
| 复购率 | 统计周期内，下单次数大于1的访客占所有下单访客的百分比。 |
| 营销活动带来的销售额  | 外部营销活动（推广链接带utm参数）带来的销售额。
utm参数包括：utm_source、utm_medium、utm_campaign、utm_content、utm_term |
| 支付渠道统计 | 支付渠道：通过某个支付渠道下单的订单数
支付渠道占比：某支付渠道订单数/总订单数量 |
| 浏览量 | 店铺下所有页面的访客浏览量（PV） |
| 会员 | 注册会员数：指的是所选时间段内成功注册的会员总数。


消费会员数：指的是所选时间段内成功下单消费的注册会员总数。
会员消费总金额：指的是所选时间段内，注册会员所有有效订单总消费金额
消费会员客单价：所选时间段内，会员消费总金额除以消费会员数。 |
